# 🪐 FRLG デオキシス自動厳選システム (ツインシステム版)

数ヶ月後の自分へ：これはゲームボーイアドバンス（FRLG）のデオキシス色違いを、PC（画像認識）と開発ボード（Leonardo）を連携させて24時間完全自動で厳選するためのプロジェクトです。

---

## 🛠️ 1. 何をインストールしているか（依存環境）

このプロジェクトは以下の環境とライブラリで動作しています。

### ベース環境
* **Python 3.12.x**（最新すぎる3.14等はライブラリが未対応のため不可、必ず3.12系を使用すること）
* **Arduino IDE**（マイコンへのプログラム書き込み用・裏方）
* **SwitchControlLibrary (by celclow)**（Arduino用のSwitch操作ライブラリ）

### Python外部ライブラリ（画面認識・マイコン通信用）
* `opencv-python` : キャプチャーボードの映像処理用
* `numpy` : 色味（RGB値）の数値計算用
* `pyserial` : PCから開発ボードへUSBシリアル信号を送る用
* `pyautogui` : 画面の特定座標のピクセル色を取得する用
* `Pillow` : 画面スクショの内部処理用

---

## 🚀 2. 久しぶりに起動するときの全手順

環境を再構築、または起動するときは以下のステップを上から順に実行すれば動きます。

### ステップ①：Pythonライブラリの一括導入
VS Codeのターミナルを開き、以下のコマンドをコピペして実行する。
```bash
python -m pip install opencv-python numpy pyserial pyautogui Pillow

```

### ステップ②：マイコン（開発ボード）の準備

1. 開発ボード（Leonardo）をPCにUSBで繋ぐ。
2. Arduino IDEを開き、`Switch_Deoxys.ino` を書き込む。
3. ボードをSwitchドックのUSBポートに差し替える。
4. ボードとPCを別のUSBケーブルで接続する（シリアル通信の開通）。

### ステップ③：マクロの実行

1. キャプチャーボードの映像（OBS等）をPC画面に映し、デオキシスの右肩が以前特定した座標（`1343, 152`）に来るように配置する。
2. VS Codeで `deoxys_macro_proto.py` を開き、右上の **「再生ボタン（▷）」** を押す。

> 💡 **途中で強制終了したいとき：**
> VS Codeのターミナルをクリックした状態で、キーボードの **`Ctrl + C`** を押せば安全に緊急停止します。

## 🧹 3. 使わなくなったら何をアンインストールするか

厳選が終了し、PCからこのシステムを完全に削除して綺麗な状態に戻したいときは、以下の手順を行います。

### ステップ①：インストールしたPythonライブラリの全削除

VS Codeのターミナルで以下のコマンドを実行すると、今回入れたライブラリがPCから綺麗に消え去ります。

```bash
python -m pip uninstall -y opencv-python numpy pyserial pyautogui Pillow

```

### ステップ②：Python本体の削除（必要な場合のみ）

もしPCで今後一切Pythonを使わない場合は、以下を実施。

1. Windowsの「設定」➔「アプリ」➔「インストールされているアプリ」を開く。
2. 「Python 3.12」を探して「アンインストール」を実行。

---

## 🎯 補足：ターゲットデータ（通常色デオキシス）

もし画面の配置やサイズが変わって再調整が必要になった場合の備忘録。

* **監視座標 (X, Y) :** `(1343, 152)` ※フルスクリーンプレイ時
* **通常色の基準 (R, G, B) :** `(254, 132, 73)` （特徴的なオレンジ色）
* **判定ロジック :** 戦闘開始後、この座標がオレンジなら「通常」とみなして即リセット。もし時間内にオレンジにならなければ「色違いエフェクト発生中」とみなしてマクロを自動停止する。
